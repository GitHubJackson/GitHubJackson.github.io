vue组件(页面)间传值

发表于 2019-01-26 | 分类于 vue | 阅读次数：

小小的总结一下vue组件间传值吧，其实也适用与页面之间，因为vue页面也就是一个大组件。

父传子 [父组件中的子组件带参和子组件声明props]

子传父 [子组件$emit触发事件a和父组件监听事件a]

非父子

路由传参

  • query

    query传值会显示在url上，页面刷新数据不会消失，用path指定路由

  • params

    传值不会显示在url上，但页面刷新数据会消失，用name指定路由

sessionStorage

先简单区别一下localStorage和sessionStorage

  • sessionStorage数据的存储仅特定于某个会话中，也就是说数据只保持到浏览器关闭，当浏览器关闭后重新打开这个页面时，之前的存储已经被清除。
  • localStorage是一个持久化的存储，它并不局限于会话。用这个的话，有时候要记得清除或者设置定时清除缓存

eventBus

少页面小项目，用eventBus是个不错的选择。

  • 定义一个vue实例专门用于传递数据

  • 在组件A定义传递的方法名和传输内容，点击事件或钩子函数触发eventBus.emit事件

  • 组件B接收传递过来的数据

vuex

曾经有前辈说：不要为了使用vuex而去使用vuex，现在看来，确实有几分道理。因为有很多项目的组件间数据实际上并没有很多交互，用了vuex不仅增加了学习成本，还让人觉得杀鸡用牛刀。vuex适合用于组件或页面间数据交互频繁的中大型应用，可以很方便地统一管理公共数据。这里只是点一下有这个东西，可以用它实现非父子传值，具体的用法后面再总结吧，因为我也不是很熟悉这块emmm…
