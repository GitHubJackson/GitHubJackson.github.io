使用plugin – 功能扩展

通过在构建流程中注入钩子，来扩展webpack的功能

下面通过plugin将注入到bundle.js的css提取到单独的文件，而不是用style-loader将css样式嵌入head标签。这里先安装extract-text-webpack-plugin

如果使用style-loader将样式添加到js文件中，在编写样式的时候可以享受热更新的效果，如果使用extract-text-webpack-plugin 将样式提取，此组件并不支持热更新。只会重新打包但是并不会刷新页面。

热更新机制

npm run dev
运行服务器，此时默认有实时预览的功能，尝试修改css文件和js文件，会看到浏览器自动刷新；但是修改index.html却没用，原因是只有entry本身和依赖的文件才会被webpack添加到监听列表里，而index.html是脱离了js模块化系统的，所以webpack无法监听到

除了通过重新刷新整个网页实现实时预览，还可以通过模块热更新实现，在不重新加载整个网页的情况下，通过将已更新的模块替换老模块，再重新执行一次实现实时预览，相比之下速度更快。
