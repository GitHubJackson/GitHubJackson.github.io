标识符

  • 合法：str/$str/我是str/_str
  • 非法：1str/#str/s-tr/s#tr/false

严格模式

"use strict"

在全局头部声明[全局]或者函数头部声明[局部]，nodejs开发建议全局使用严格模式

  • 严格的变量声明，若不加严格模式，当不给变量声明var时，默认被定义为全局变量
  • 禁止动态绑定，不得使用with、eval作用域，提高执行效率
  • 不能删除变量，即不能delete 变量
  • 不得重名：属性名、参数名
  • 禁止使用八进制数字
  • 保留字不得用作标识符

语句格式

  • 建议使用分号结尾，提升运行速率
  • 大括号可用可不用时建议使用

数据类型

数据类型的分类

  • 基本数据类型：undefined/null/String/Number/Boolean/symbol(ES6)
    • Number(null)=0,Number(undefined)=NaN
    • null是个空对象
    • Boolean()将其他类型转换为布尔值，空字符、0、null、NaN、undefined会被转换成false
    • 浮点数占用空间是整数两倍
    • 八进制：010，严格模式无法使用；十六进制：0x10
    • Number.MAX_VALUE/MIN_VALUE指js中能表示的最大值/最小值，isFinite()
  • 基本类型转换
  • 复杂数据类型：Object
    • 初始化，new Object()/{}
    • 对象方法
  • typeof(操作符) 检查数据类型
  • 不能创建自定义数据类型
