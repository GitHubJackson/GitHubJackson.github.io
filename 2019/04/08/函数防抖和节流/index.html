函数防抖和节流

发表于 2019-04-08 | 分类于 Javascript | 阅读次数：

前端开发过程中会遇到一些频繁的事件触发函数，比如：滑动事件、鼠标和键盘事件、搜索框输入匹配、resize等等，如果不限制函数的执行次数，会导致函数触发频率过高，使页面出现延迟、卡顿的现象。

函数防抖或节流

防抖和节流的作用都是防止函数多次调用。区别在于，假设一个用户一直触发这个函数，且每次触发函数的间隔小于wait，防抖的情况下只会调用一次，而节流的 情况会每隔一定时间（参数wait）调用函数。

防抖(debounce)

让某一段程序在指定的事件之后触发

节流(throttle)

让一段程序在规定的时间内只执行一次

应用场景

其他场景

  • 搜索框input事件
    例如要支持输入实时搜索可以使用节流方案（间隔一段时间就必须查询相关内容），或者实现输入间隔大于某个值（如500ms），就当做用户输入完成，然后开始搜索
  • 页面resize事件
    常见于需要做页面适配的时候。需要根据最终呈现的页面情况进行dom渲染（这种情形一般是使用防抖，因为只需要判断最后一次的变化情况）
