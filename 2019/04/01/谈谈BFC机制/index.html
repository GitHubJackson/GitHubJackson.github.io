谈谈BFC机制

发表于 2019-04-01 | 分类于 css | 阅读次数：

BFC，Block Formatting Contexts(块级格式化上下文)，可理解为一种具有特性的盒子，在BFC盒子里的元素会有些特性，但不论盒子内元素如何改变，都不会影响到外部元素。

触发条件

  • body、根元素html
  • float：除 none 以外的值
  • position：(absolute、fixed)
  • display： inline-block、table-cells、flex
  • overflow：除了 visible 以外的值 (hidden、auto、scroll)
  • 常用：overflow:hidden

BFC特性

  • 在垂直方向上的margin会发生重叠
  • 在页面上是一个独立的容器,外面的元素和里面的元素互不影响
  • BFC元素不会和浮动的元素重叠
  • 计算BFC元素的高度时,里面浮动元素的高度也会参与计算

BFC用途

解决上下外边距重叠的问题

上边div的margin-bottom和下边div的margin-top同时设置，会产生边距重叠的现象。

清除浮动

父级元素添加overflow:hidden，使其成为一个BFC盒子，在计算高度的时候会将浮动的子元素计入，达到清除浮动的效果
